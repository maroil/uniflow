# Uniflow CDP

An open-source Customer Data Platform (CDP) you can self-host in your own AWS account.

## Features

- **Event Collection** — Segment-compatible HTTP API (`track`, `identify`, `page`, `group`)
- **Identity Resolution** — Anonymous ID → known user ID linking with identity graph
- **Unified Customer Profiles** — Merged profiles with traits and event history in DynamoDB
- **Segmentation** — Rule-based audience segments with scheduled Athena evaluation
- **Destinations** — Webhook and S3 export (plugin architecture for community connectors)
- **Admin UI** — React dashboard: sources, destinations, profile explorer, segments
- **CLI** — `uniflow init / deploy / upgrade / status / destroy`

## Architecture

```
Client SDK → API Gateway → Lambda (validate) → Kinesis Stream
                                               → Kinesis Firehose → S3 (raw)
Kinesis Stream → Lambda (processor) → DynamoDB (profiles + identity graph)
                                    → SQS (destination fan-out)
EventBridge (hourly) → ECS Fargate (audience builder) → Athena → DynamoDB (segments)
Cognito → API Gateway → Lambda (management API)
Next.js (static export) → S3 → CloudFront
```

## Quick Start

```bash
# Install CLI
npm install -g uniflow

# Interactive setup
uniflow init

# Deploy to AWS
uniflow deploy

# Check health
uniflow status
```

## Repository Structure

```
uniflow/
├── infra/                    # CDK stacks + constructs (@uniflow/cdk)
│   └── src/
│       ├── stacks/UnifowStack.ts
│       └── constructs/
│           ├── StorageConstruct.ts      # DynamoDB + S3 + Kinesis + Firehose
│           ├── IngestionConstruct.ts    # API Gateway + Lambda
│           ├── ProcessingConstruct.ts   # Kinesis consumer Lambda + SQS
│           ├── AudienceConstruct.ts     # ECS Fargate + EventBridge Scheduler
│           └── AdminConstruct.ts        # Cognito + Lambda + CloudFront
├── services/
│   ├── ingest/               # Ingest Lambda (validate → Kinesis)
│   ├── processor/            # Kinesis consumer (identity + profiles)
│   ├── audience-builder/     # Fargate task (Athena → segment membership)
│   └── management-api/       # CRUD Lambda (sources/destinations/segments)
├── connectors/
│   ├── sdk/                  # BaseConnector interface
│   ├── webhook/              # HTTP webhook connector
│   └── s3-export/            # S3 dump connector
├── libs/
│   ├── event-schema/         # Zod schemas for all event types
│   ├── identity/             # Identity resolution logic
│   └── logger/               # Structured logging
├── cli/                      # uniflow CLI (published as `uniflow` on npm)
├── sdk/js/                   # @uniflow/js browser + Node.js SDK
├── ui/                       # Next.js 16 admin dashboard
├── docker/                   # docker-compose + LocalStack for local dev
└── examples/                 # Sample CDK app + tracking scripts
```

## Local Development

```bash
# Prerequisites: Docker, Node.js 20+, pnpm 9+

# Clone and install
git clone https://github.com/your-org/uniflow
cd uniflow
pnpm install

# Start LocalStack (DynamoDB, S3, Kinesis, SQS)
docker compose -f docker/docker-compose.yml up -d localstack

# Send test events
npx ts-node examples/send-events.ts

# Run all tests
pnpm test

# Build all packages
pnpm build
```

## DynamoDB Single-Table Design

| Entity | PK | SK |
|---|---|---|
| Profile | `PROFILE#<userId>` | `META` |
| Event | `PROFILE#<userId>` | `EVENT#<ts>#<id>` |
| Identity | `ANON#<anonymousId>` | `IDENTITY` |
| Segment | `SEGMENT#<id>` | `META` |
| Segment Member | `SEGMENT#<id>` | `MEMBER#<userId>` |
| Source | `SOURCE#<id>` | `META` |
| Destination | `DEST#<id>` | `META` |
| Migration | `MIGRATION#<id>` | `META` |

## Connector SDK

Build your own connector:

```typescript
import { BaseConnector, type ConnectorEvent, type ConnectorResult } from '@uniflow/connector-sdk';
import { z } from 'zod';

const MyConfigSchema = z.object({ apiKey: z.string() });

export class MyConnector extends BaseConnector<z.infer<typeof MyConfigSchema>> {
  readonly metadata = {
    id: 'my-connector',
    name: 'My Connector',
    description: 'Send events to My Service',
    configSchema: MyConfigSchema,
  };

  async handle(event: ConnectorEvent, config: z.infer<typeof MyConfigSchema>): Promise<ConnectorResult> {
    // Send event.event to your service
    return { success: true };
  }
}
```

Publish as `@uniflow/connector-<name>` for community use.

## Configuration

`uniflow.config.yaml` (version-controlled, generated by `uniflow init`):

```yaml
version: "0.1"
region: us-east-1
adminEmail: admin@acme.com
retentionDays: 90
connectors:
  - webhook
  - s3-export
stackName: UnifowStack
```

## Technology Stack

| Layer | Technology |
|---|---|
| IaC | AWS CDK (TypeScript) |
| Ingest compute | Lambda + API Gateway HTTP API |
| Long-running compute | ECS Fargate |
| Event buffer | Kinesis Data Streams (7-day) |
| Hot store | DynamoDB (single-table, PAY_PER_REQUEST) |
| Cold store + analytics | S3 + Athena + Glue Catalog |
| Auth | Cognito User Pool |
| Delivery reliability | SQS + DLQ |
| UI | Next.js 15 + Tailwind CSS v4 |
| Monorepo | pnpm + Turborepo |
| Local dev | docker-compose + LocalStack |
